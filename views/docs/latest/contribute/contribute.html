Contribute

We greatly value and encourage all kinds of contributions. Before getting started, please review the relevant section to ensure a smooth process for everyone and to assist the maintainers in managing contributions effectively. The community eagerly awaits your input!

If you love the project but don’t have the time to contribute, that’s absolutely fine! There are other simple ways to support and show your appreciation, which we’d greatly value:

  • Give the project a star
  • Share it on Twitter
  • Add a reference to it in your project’s README
  • Mention it at local meetups or share it with your friends and colleagues

I Have a Question

We assume you’ve gone through the available documentation before raising a question.

It’s recommended to search for existing issues before asking a question, as you may find helpful information. If you come across a relevant issue but need more clarification, feel free to add your question there. Additionally, consider reaching out to the community on the rumblesheetjs Discord server for further support.

If you still need clarification after searching, we suggest the following:

  • Open a new issue.
  • Include as much detail as possible about the issue you’re encountering.
  • Provide relevant information such as project version, browser, and operating system.

We (or someone from the community) will address the issue as soon as possible.
